Модульна карта курсу

1. Мотивація

Сфера застосування автоматизації. Як код керує Штучним Інтелектом (ChatGPT), GameDev та Веб-сервісами.

2. Синтаксис

Граматичні правила мови. Чутливість до регістру, структура рядків та керування лапками.

3. Змінні

Оперативна пам'ять комп'ютера. Створення посилань на об'єкти, оператор присвоєння (=) та базові типи даних.

4. Ввід/Вивід

Побудова інтерактивного діалогу. Функції input() та print(), динамічне перетворення типів даних.

5. Розгалуження

Керування логікою алгоритму через if/else. Булева алгебра (True/False) та обов'язкові структурні відступи.

6. Цикли

Автоматизація рутини. Цикли for та робота з генератором діапазонів чисел range().

7. Списки

Концепція індексованих масивів. Збереження великих груп інформації, нульова індексація та ітерація.

8. Функції

Принцип DRY (Не повторюйся). Оголошення def, ізоляція логіки, вхідні аргументи та повернення результатів.

9. Проєкт

Фінальний виклик. Інтеграція всіх вивчених концепцій у комплексну архітектуру гри-вікторини з підрахунком очок.

Теорія: Правила побудови інструкцій

Python є **case-sensitive** (чутливим до регістру літер). Це означає, що команди `print()`, `Print()`, `PRINT()` та `prInT()` — це абсолютно різні речі для комп'ютера. Лише перший варіант є вбудованою командою, інші викликають критичну помилку `NameError`.

Текстові рядки обов'язково оточуються парними лапками. Можна використовувати як подвійні "...", так і одинарні '...'. Вони потрібні для того, щоб інтерпретатор відрізняв команду мови від звичайного людського тексту. Символ решітки # позначає **коментар** — цей текст комп'ютер повністю ігнорує, він написаний виключно для людини-програміста, щоб пояснити складні місця у програмі.

Етап 3. Архітектура пам'яті: Концепція змінних

Що пояснити? Оператор присвоєння (=), виділення комірок оперативної пам'яті (RAM), типи String та Integer.
Кому пояснити? Учням, які плутають знак рівності в математиці зі знаком присвоєння у кодингу.
Як пояснити? Метафора: Змінна — це коробка в пам'яті комп'ютера. Назва коробки — це ім'я змінної, а вміст коробки — її значення.

Теорія: Динамічне виділення пам'яті

У комп'ютері є оперативна пам'ять (RAM), яка складається з мільярдів дрібних електронних комірок. Коли ми пишемо player_score = 150, ми даємо вказівку комп'ютеру: *«Знайди вільне місце в пам'яті, поклади туди число 150 і наклей на цю комірку ярлик з іменем player_score»*.

[Оперативна пам'ять Комп'ютера] ┌──────────────────┬──────────────────┐ │ Ярлик: name │ Значення: "Oleg" │ <-- Тип: String (Текст) ├──────────────────┼──────────────────┤ │ Ярлик: score │ Значення: 150 │ <-- Тип: Integer (Ціле число) └──────────────────┴──────────────────┘

Знак «=» в програмуванні — це **оператор присвоєння**, а не математична рівність. Він працює справа наліво: обчислює значення з правого боку і записує результат у змінну, яка стоїть ліворуч. Якщо ми напишемо score = score + 10, комп'ютер спочатку візьме старе значення з коробки, додасть 10 і покладе новий результат назад.

Теорія: Конвертація типів у потоках

Програма стає інтерактивною, коли вона вміє слухати користувача. Функція input("Підказка") призупиняє хід виконання програми, виводить на екран діалогове вікно та очікує, поки користувач введе дані й натисне Enter.

Проте існує критично важливий нюанс: **`input()` завжди сприймає введені дані як текстовий рядок (String)**. Якщо користувач введе число `14`, Python запише його в пам'ять як текст `"14"`. Спроба додати до нього одиницю викличе помилку `TypeError`. Щоб перетворити текст на справжнє математичне ціле число, його потрібно загорнути у функцію трансформації: int(). Цей процес називається **явним приведенням типів** (Type Casting).

Теорія: Булева логіка та блоки відступів

Умовний оператор дозволяє програмі приймати рішення самостійно залежно від обставин. В основі лежить перевірка виразу, який може бути або правдою (**True**), або брехнею (**False**). Для порівняння значень використовуються спеціальні знаки: == (перевірка на рівність), > (більше), < (менше), != (не дорівнює).

На відміну від багатьох інших мов програмування, де блоки коду виділяються фігурними дужками, в Python для цього використовуються **структурні відступи (4 пробіли або клавіша Tab)**. Двокрапка в кінці інструкції `if` або `else` каже інтерпретатору: *«Увага, далі йде підпорядкований блок коду»*.

if age >= 18: ──── print("Доступ дозволено") # Виконається лише якщо True else: ──── print("Доступ заблоковано") # Виконається лише якщо False

Етап 6. Алгоритмічна автоматизація: Циклічні процеси

Що пояснити? Керування повтореннями через for, ітераційну змінну, роботу параметрів генератора range(start, stop).
Кому пояснити? Учням, які копіюють блоки коду вручну для виконання однакових дій.
Як пояснити? Функція range(1, 6) створює числовий конвеєр. Цикл бере перше число, виконує з ним код, повертається на початок, бере друге число і так далі, поки конвеєр не закінчиться.

Теорія: Конструкція циклу For та межа ітерацій

Цикли потрібні для того, щоб змусити комп'ютер виконувати рутинну роботу миттєво. Замість написання десяти окремих рядків print(), ми описуємо дію один раз всередині циклу.

Конструкція for i in range(1, 6): розшифровується так: створи змінну-лічильник `i` та запускай блок коду з відступом для кожного числа з діапазону. **Важливе правило функції range(start, stop): вона ніколи не включає в себе останнє число (stop)**. Тобто `range(1, 6)` згенерує числа 1, 2, 3, 4, 5. Число 6 використано як стоп-сигнал, на якому цикл зупиняє роботу.

Етап 8. Модульне проектування: Процедури та Функції

Що пояснити? Оголошення def, локальну область видимості змінних, аргументи, оператор повернення return.
Кому пояснити? Учням, які пишуть один велетенський масив коду, де важко знайти помилку.
Як пояснити? Функція — це міні-завод всередині програми. Вона приймає сировину (аргументи), виконує закриту роботу і видає готовий продукт (через return). Все, що відбувається всередині заводу, приховано від решти світу.

Теорія: Принцип DRY та ізоляція коду

Під час написання великих програм певні послідовності дій починають повторюватися. Копіювання однакових шматків коду призводить до роздування файлів та появи багів. Професійні розробники використовують принцип **DRY (Don't Repeat Yourself)** — Не повторюйся.

За допомогою ключового слова def ми можемо створити власну команду (функцію), дати їй унікальне ім'я та описати її логіку один єдиний раз. У дужках вказуються **параметри** — змінні, які функція очікує отримати ззовні для обробки. Оператор return повертає результат роботи функції назад у головний код програми.
